feign调用源码解析1

发表于2022-12-07|更新于2022-12-10|spring cloud
|阅读量:

feign

feign是netflix封装的基于rpc的远程调用架构，底层使用okhttp等工具进行http调用，默认使用ribbon进行负载均衡。

先说结论

feign利用配置类，在项目启动时，扫描@FeignClient注解，通过动态代理的方式，为其生成对应的调用方法。

spring.factories

该文件位于org.springframework.cloud.spring-cloud-openfeign-core下，提供了feignClient所需的相关配置类和client

以上是spring-cloud-openfeign-core包下的spring.factories的内容，显而易见，该spring.factories中只包含自动装配的配置类，接下来依次说明

FeignRibbonClientAutoConfiguration

该配置类用于构建重试相关的配置类

该方法注册了两个bean

CachingSpringLoadBalancerFactory

当RetryTemplate不存在时使用单参构造CachingSpringLoadBalancerFactory，
否则使用双参构造

Request

返回LoadBalancerFeignClient中的DEFAULT_OPTIONS

以上是该配置类本身引用的内容，接下来讲解import的三个配置类

HttpClientFeignLoadBalancedConfiguration

该类会新建一个ApacheHttpClient，并用ApacheHttpClient构建一个LoadBalancerFeignClient并放入容器。该类import了HttpClientFeignConfiguration，看看HttpClientFeignConfiguration

HttpClientFeignConfiguration

在之前版本中，HttpClientFeignLoadBalancedConfiguration直接使用
@Configuration注解配置，而非@Import了新的配置类

该方法先注入了一个HttpClientConnectionManager，然后会根据feign.compression.response.enabled配置的值，注入customHttpClient（为true时，注入自定义http客户端）或httpClient（默认，为false或不存在该配置时，注入）

OkHttpFeignLoadBalancedConfiguration

内容同1.1，将ApacheHttpClient换成了OkHttpClient

DefaultFeignLoadBalancedConfiguration

内容同1.1，将ApacheHttpClient换成了Client下的子类Default

FeignHalAutoConfiguration

该方法为json转换，非必须

FeignAutoConfiguration

该方法主要通过配置类，为三种不同类型的feignclient配置对应需要注入的配置类。

@EnableFeignClient

第一部分的起点在spring.factories自动装配，会将需要构建feignclient的所需类全部注入。接下来这部分的入口则在@EnableFeignClient中,该注解会使用@Import引入FeignClientsRegistrar配置类，接下来就从这个类开始。
FeignClientsRegistrar类实现了ImportBeanDefinitionRegistrar接口，重写了registerBeanDefinitions方法，动态注入了相关类

该方法主要有两个方法

  • registerDefaultConfiguration 将@EnableFeignClients中定义defaultConfiguration属性下的类，包装成FeignClientSpecification，注册到Spring容器
  • registerFeignClients 注册feignclient

该方法主要流程如下，根据配置扫描注解，调用registerFeignClient方法注册对应的FeignClientFactoryBean

接下来则是关键。在spring的启动流程中，spring会使用bean的工厂类对bean进行初始化时的处理。这里，则会使用FeignClientFactoryBean的getObject方法

最终会调用targeter.target方法，而该targeter的实现类为Feign，实际调用为Feign的target方法

而newInstance方法则在ReflectiveFeign中实现

该方法最终会使用SynchronizedMethodHandler增强feignclient的调用方法
